# 위플랩 룰렛 듀얼 타이머

위플랩 룰렛 후원 결과를 감지해서 OBS 타이머에 자동 적용하는 로컬 감지형 타이머입니다.

현재 버전은 **자동화 베타**입니다. 로컬 감지 프로그램이 위플랩 후원알림 페이지를 열고, 화면에 나타나는 룰렛 결과 텍스트를 감지해 OBS 타이머로 전송합니다.

## 다운로드

가장 쉬운 방법은 저장소에 올라와 있는 `weflab-roulette-timer.zip` 파일을 받는 것입니다.

1. 저장소 파일 목록에서 `weflab-roulette-timer.zip`을 클릭합니다.
2. 오른쪽 위 `Download raw file` 버튼을 눌러 다운로드합니다.
3. 압축을 풀고 `start_monitor.exe`를 실행합니다.

릴리즈 페이지를 사용하지 않아도 위 zip 파일만 받으면 바로 사용할 수 있습니다.

## 주요 기능

- `멧돼지`, `돼지` 타이머 2개 독립 실행
- 위플랩 룰렛 결과 자동 감지
- 타이머 이름 변경 가능
- `+30분`, `-1시간`, `돼지 +20분` 같은 입력 자동 계산
- 입력 성공 시 타이머 자동 시작
- OBS 브라우저 소스로 사용 가능
- 조작 패널 숨김 가능
- 위플랩 후원알림 URL 연결/디버그 확인 기능 포함

## 파일 구성

- `obs_timer.html`: 실제 타이머 파일
- `monitor.js`: 위플랩 룰렛 결과를 감지하는 로컬 프로그램
- `start_monitor.exe`: 더블클릭 실행용 프로그램
- `start_monitor.bat`: 감지 프로그램 실행 파일
- `사용법.txt`: 간단 사용법

## 실행 전 준비

이 프로그램은 감지 로직을 Node.js로 실행합니다. 실행 PC에 Node.js가 설치되어 있어야 합니다.

Node.js 다운로드:

```text
https://nodejs.org/
```

## 자동화 사용 방법

1. `start_monitor.exe`를 실행합니다.
2. 위플랩 후원알림 URL을 입력합니다.
3. 첫 번째 타이머 이름과 두 번째 타이머 이름을 입력합니다.
   - 기본값은 `멧돼지`, `돼지`입니다.
4. 프로그램이 알려주는 OBS 타이머 주소를 복사합니다.

```text
http://127.0.0.1:17354/timer
```

5. OBS에서 `소스 추가` → `브라우저`를 선택합니다.
6. URL에 위 주소를 넣습니다.
7. 위플랩 룰렛 테스트 또는 실제 룰렛 후원을 실행합니다.

감지에 성공하면 콘솔에 아래처럼 표시되고, OBS 타이머가 자동으로 적용됩니다.

```text
[적용 전송] 돼지 +30분
```

## 수동 입력도 가능

자동 감지가 안 되는 상황에서는 OBS 타이머 패널에서 직접 입력할 수도 있습니다.

예시:

```text
+30분
-1시간
돼지 +20분
멧돼지 +1시간
```

입력창에 이름 없이 시간만 적은 경우:

```text
+30분
```

이때는 `멧돼지 적용` 또는 `돼지 적용` 버튼을 눌러 적용할 대상을 선택합니다.

여러 결과를 한 번에 넣을 수도 있습니다.

```text
멧돼지 +1시간
돼지 -30분
```

이 경우 `전체 적용` 버튼을 누르면 각각의 타이머에 적용됩니다.

## 조작 패널 숨기기

OBS 화면에 조작 패널이 보이지 않게 하려면 URL 뒤에 아래 값을 붙입니다.

```text
?controls=0
```

예시:

```text
obs_timer.html?controls=0
```

패널을 다시 보고 싶으면 URL에서 `?controls=0`을 제거하거나, 화면에서 `H` 키를 누릅니다.

## 위플랩 URL 연결 테스트

위플랩 후원알림 URL을 입력하면 연결 상태를 확인할 수 있습니다.

현재 확인된 연결 메시지 예시:

```json
{"uniConnect":true}
```

이 메시지는 룰렛 결과가 아니라 위플랩 연결 확인 신호입니다.

실제 룰렛 후원 시 디버그 창에 다른 메시지가 뜨면, 그 내용을 기반으로 자동연동 기능을 추가할 수 있습니다.

## 현재 한계와 확인할 점

- 위플랩 화면의 텍스트를 감지하는 방식이라, 위플랩 알림 문구 구조가 바뀌면 감지 규칙 조정이 필요할 수 있습니다.
- 첫 실행 시 Windows 방화벽 또는 브라우저 보안 확인 창이 뜰 수 있습니다.
- `start_monitor.exe` 실행 중에는 열린 Edge/Chrome 창을 닫지 않는 것이 좋습니다.
- OBS에는 로컬 파일이 아니라 `http://127.0.0.1:17354/timer` 주소를 넣는 것을 권장합니다.

## 배포 상태

자동화 베타 버전입니다.
